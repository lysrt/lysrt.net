Using Caddy

Wed, Jan 3, 2018

When self hosting my website, most of the time I had to choose between Apache HTTP Server and NGINX.

For a simple web site, Apache would be more than enough all the time.

Comes the need to host an API? No worries, Apache can proxy several endpoints on a single computer. Have several Domain Names and this is it. But what if you need to do something like: mysite.com/api/? NGINX reverse proxy was the way to go for me.

I then started to work and have less time to spend to maintain NGINX or Apache configuration files, and I forgot about the upgrade procedure every time.

Meet Caddy.

lysrt.net {
    gzip
    root /path/to/lysrt.net/
    tls my@email.com

    proxy /api localhost:8080 {
        without /api
        transparent
    }

    log stdout
    errors {
        log stderr
        404 404.html
    }
}

And this is it! no more config for a running full TLS server, serving compressed pages, with transparent reverse proxy API, proper logging, and a custom 404 page (why not?)
